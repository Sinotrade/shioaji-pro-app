## v0.1.29 — Windows 打字失效與模式切換凍結修正

### 修正
- **Windows：對話框打不了字**：視窗明明在前景、滑鼠也正常，但鍵盤怎麼打都沒反應，只能重啟 app。根因是原生視窗與 WebView2 的鍵盤焦點失同步 — 現在 app 會在視窗獲得焦點或點擊頁面時自動檢查並把鍵盤焦點要回來，即使發生失同步也會立即自癒，不再需要重啟。
- **模擬↔正式切換瞬間卡住**：伺服器停止、埠掃描等桌面指令原本在主執行緒同步執行（netstat/taskkill 子程序加上最多數秒的等待迴圈），切換模式時整個視窗凍結 — 這同時也是鍵盤焦點錯亂的觸發源之一。已全部移到背景執行緒，切換過程 UI 保持流暢。
- 從系統匣喚回主視窗時，明確把鍵盤焦點交給頁面，避免喚回後打不了字。
